---
title: "croc environment variables"
description: "Reference for all environment variables recognized by croc, including the shared secret, relay address, relay password, and proxy settings."
---

croc reads several environment variables that let you configure transfer settings without exposing them on the command line. This is especially important on Linux and macOS, where command-line arguments are visible to all users in the process list via tools like `ps`. Using environment variables keeps sensitive values out of process listings.

## Variables

<ParamField path="CROC_SECRET" type="string">
  The code phrase (shared secret) used to authenticate and encrypt the transfer. Setting this variable on the sender and receiver sides is the recommended way to specify a custom code phrase on Linux and macOS.

  Equivalent CLI flag: none (replaces `--code` for send and the positional code argument for receive on non-Windows systems)

  Example: `CROC_SECRET="apple-mango-river"`
</ParamField>

<ParamField path="CROC_RELAY" type="string">
  Address of the relay server, overriding the built-in default of `croc.schollz.com`. Both sender and receiver must point to the same relay.

  Equivalent CLI flag: `--relay`

  Example: `CROC_RELAY="myrelay.example.com:9009"`
</ParamField>

<ParamField path="CROC_RELAY6" type="string">
  IPv6 address of the relay server, overriding the built-in default of `croc6.schollz.com`.

  Equivalent CLI flag: `--relay6`

  Example: `CROC_RELAY6="myrelay6.example.com:9009"`
</ParamField>

<ParamField path="CROC_PASS" type="string">
  Password for the relay server. Must match the password the relay was started with. Overrides the built-in default of `pass123`.

  Equivalent CLI flag: `--pass`

  Example: `CROC_PASS="myrelaypassword"`
</ParamField>

<ParamField path="SOCKS5_PROXY" type="string">
  SOCKS5 proxy address used for all relay connections. Applies to both `croc send` and receive operations.

  Equivalent CLI flag: `--socks5`

  Example: `SOCKS5_PROXY="127.0.0.1:9050"`
</ParamField>

<ParamField path="HTTP_PROXY" type="string">
  HTTP proxy address used for all relay connections.

  Equivalent CLI flag: `--connect`

  Example: `HTTP_PROXY="http://proxy.example.com:8080"`
</ParamField>

## Usage examples

### Receiving a file (Linux/macOS)

On Linux and macOS, `CROC_SECRET` is the preferred way to supply the code phrase. Passing the code phrase as a positional argument is blocked in the default (non-classic) mode to prevent process list exposure.

```bash
CROC_SECRET="word-word-word" croc
```

### Sending a file with a custom code phrase (Linux/macOS)

```bash
CROC_SECRET="word-word-word" croc send file.txt
```

### Using a self-hosted relay

```bash
CROC_RELAY="myrelay.example.com:9009" croc send file.txt
```

On the receiving end, set the same variable:

```bash
CROC_RELAY="myrelay.example.com:9009" CROC_SECRET="word-word-word" croc
```

### Using a password-protected relay

```bash
CROC_PASS="mypassword" croc --relay "myrelay.example.com:9009" send file.txt
```

### Routing through Tor

```bash
SOCKS5_PROXY="127.0.0.1:9050" croc send file.txt
```

## Security note: CROC_SECRET vs --code

<Warning>
  On Linux and macOS, avoid passing the code phrase as a command-line argument (`croc send --code mycode file.txt` or `croc mycode`). The argument is visible to all local users via `ps aux` or `/proc`. This vulnerability is tracked as [CVE-2023-43621](https://nvd.nist.gov/vuln/detail/CVE-2023-43621).

  Use `CROC_SECRET` instead, or enable classic mode only on single-user systems with `croc --classic`.
</Warning>

On Windows, the process list does not expose environment in the same way, so passing the code phrase as a command-line argument is less of a concern. croc applies the `CROC_SECRET` requirement only on non-Windows platforms when classic mode is disabled.
