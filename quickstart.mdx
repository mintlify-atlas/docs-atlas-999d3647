---
title: "Quickstart"
description: "Transfer your first file with croc in under a minute. Install croc, send a file, and receive it on another machine using a short code phrase."
---

croc makes file transfer as simple as running one command on each side. The sender gets a code phrase; the receiver uses it to connect. All data is end-to-end encrypted via PAKE before it reaches the relay.

<Steps>
  <Step title="Install croc">
    Install croc using your preferred method. The quickest way on macOS or Linux:

    ```bash
    curl https://getcroc.schollz.com | bash
    ```

    On macOS with Homebrew:

    ```bash
    brew install croc
    ```

    On Windows with Scoop:

    ```bash
    scoop install croc
    ```

    For all platforms and package managers, see the [installation guide](/installation).
  </Step>
  <Step title="Send a file">
    On the sending machine, run:

    ```bash
    croc send file.txt
    ```

    croc prints a code phrase and waits for the receiver to connect:

    ```
    Sending 'file.txt' (1.2 MB)
    Code is: grove-angle-mauve
    ```

    The code phrase is also copied to your clipboard automatically. Share it with the recipient through any channel — chat, email, or read it aloud.

    To send a folder or multiple files:

    ```bash
    croc send folder/
    croc send file1.txt file2.txt file3.txt
    ```
  </Step>
  <Step title="Receive the file">
    On the receiving machine, use the code phrase to start the transfer.

    <Tabs>
      <Tab title="Linux / macOS">
        On Linux and macOS, pass the code phrase as an environment variable to avoid [leaking the secret via the process list](https://nvd.nist.gov/vuln/detail/CVE-2023-43621) (CVE-2023-43621):

        ```bash
        CROC_SECRET=grove-angle-mauve croc
        ```

        Alternatively, run `croc` with no arguments and enter the code phrase when prompted:

        ```bash
        croc
        # Enter receive code: grove-angle-mauve
        ```
      </Tab>
      <Tab title="Windows">
        On Windows, pass the code phrase directly as an argument:

        ```bash
        croc grove-angle-mauve
        ```
      </Tab>
    </Tabs>

    croc connects to the sender via the relay, performs the PAKE handshake, and downloads the file:

    ```
    Receiving 'file.txt' (1.2 MB)
    grove-angle-mauve
    100% |████████████████████| (1.2/1.2 MB, 4.6 MB/s)
    ```
  </Step>
</Steps>

## Code phrase and security

The code phrase (e.g. `grove-angle-mauve`) is a short random string generated by the sender. It seeds a [Password Authenticated Key Exchange (PAKE)](https://en.wikipedia.org/wiki/Password-authenticated_key_agreement) protocol between the two machines, which derives a shared encryption key without ever transmitting the password over the network. The relay at `croc.schollz.com:9009` brokers the connection but never sees plaintext data.

<Warning>
  On Linux and macOS, passing the code phrase as a command-line argument (e.g. `croc grove-angle-mauve`) exposes it in the system process list, where other local users can read it. Always use `CROC_SECRET=<phrase> croc` on multi-user systems. On single-user systems, you can permanently restore the classic behavior with `croc --classic`.
</Warning>
