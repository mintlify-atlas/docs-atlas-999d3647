croc is a command-line tool that makes it simple to securely transfer files and folders between any two computers. It uses Password Authenticated Key Exchange (PAKE) to establish end-to-end encrypted connections through a relay — no open ports, no server configuration, just a single code phrase shared between sender and receiver.

## Why croc?

croc is the only CLI file-transfer tool that combines all of the following:

- **End-to-end encryption** using PAKE — only sender and receiver can read the data
- **Works between any two computers** via a public relay (no port-forwarding needed)
- **Cross-platform** — Windows, Linux, macOS, and more
- **Resumable transfers** — interrupted transfers pick up where they left off
- **Multiple files and folders** in a single command
- **IPv6-first** with automatic IPv4 fallback
- **Proxy support** for SOCKS5 and HTTP proxies (including Tor)

## How it works

<Steps>
  <Step title="Sender runs croc send">
    The sender runs `croc send <file>`. croc generates a short code phrase and displays it.
  </Step>
  <Step title="Share the code phrase">
    The sender shares the code phrase with the recipient — via chat, email, or any channel.
  </Step>
  <Step title="Recipient runs croc with the code">
    The recipient runs `croc <code-phrase>` on their machine and the encrypted transfer begins.
  </Step>
</Steps>
